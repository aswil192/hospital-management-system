HOSPITAL MANAGEMENT SYSTEM - PROJECT SUMMARY
=============================================================================

PROJECT OVERVIEW:
----------------
A complete, fully functional Hospital Management System built with PHP and 
MySQL featuring three user roles (Patient, Doctor, Admin) with comprehensive 
functionality for managing appointments, billing, prescriptions, and more.

TECHNOLOGY STACK:
----------------
- Backend: PHP (Procedural style with some OOP concepts)
- Database: MySQL
- Frontend: HTML5, CSS3, Bootstrap 5
- JavaScript: jQuery, WOW.js, Owl Carousel
- Security: Password hashing, Prepared statements, Session management

DATABASE STRUCTURE:
==================

TABLES (9):
----------
1. users - Main authentication (id, name, email, password, role, phone, address, status, timestamps)
2. patients - Patient info (id, user_id, dob, gender, blood_group, medical_history)
3. doctors - Doctor info (id, user_id, specialization, license, qualification, experience, salary, join_date, availability)
4. appointments - Bookings (id, patient_id, doctor_id, date, time, status, symptoms, notes)
5. medicines - Inventory (id, name, description, price, stock, manufacturer, expiry)
6. prescriptions - Doctor prescriptions (id, appointment_id, medicine_id, dosage, frequency, duration, instructions)
7. bills - Patient bills (id, patient_id, doctor_id, appointment_id, amount, description, status, due_date)
8. payments - Payment records (id, bill_id, amount, payment_method, payment_date, status, transaction_id)
9. salaries - Doctor salaries (id, doctor_id, amount, payment_month, payment_date, status, notes)
